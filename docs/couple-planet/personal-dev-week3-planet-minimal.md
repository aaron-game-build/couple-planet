# 情侣星球个人开发 Week3 执行清单（星球最小展示）

## 本周目标

做一个“看得懂、能反馈”的星球首页，不做复杂视觉系统。

验收口径：用户能直观看到互动行为带来的星球变化。

## 功能最小范围

星球首页只展示 3 个数据：

1. 今日互动数
2. 在途信封数
3. 总能量

状态只保留 3 档（先不做 Bloom）：

- `Dormant`：24h 无互动
- `Warm`：有即时聊天
- `Active`：有马车信封行为

## 最小 API 清单

- `GET /planet/home`
- `GET /planet/energy/ledger?limit=20`
- `POST /planet/state/evaluate`（可选，调试用）

## 页面最小清单

- 星球首页（可先用静态 2D 背景 + 文案状态）
- 能量流水弹层（或二级页）

## 交互最小清单

- 点击星球：显示“今日互动摘要”
- 点在途信封数：跳转旅程列表
- 点总能量：打开能量流水

## 开发顺序（个人开发版）

1. 先聚合后端 `planet/home` 数据
2. 前端做一页式星球看板
3. 把聊天和信封事件接到状态更新
4. 做基础能量流水展示

## 每日可执行任务（7 天）

- Day 1：`planet/home` 聚合接口
- Day 2：状态计算逻辑
- Day 3：能量流水接口
- Day 4：星球首页 UI 骨架
- Day 5：状态样式联动（Dormant/Warm/Active）
- Day 6：跳转和交互完善
- Day 7：联调 + 可用性修复

## 本周验收 checklist

- [ ] 星球页冷启动可进入
- [ ] 聊天/信封行为能驱动状态变化
- [ ] 今日互动数、在途数、总能量显示正确
- [ ] 能量流水最近 20 条可查看

## 本周只看 2 个数

- 星球页次日回访率
- 星球页触发后 10 分钟内互动率
