# 情侣星球 MVP 通讯边界规范（混合模式）

## 1. 目标与设计原则

本规范定义 `即时通道` 与 `马车信封` 的职责边界，确保两者互补而非互斥。

- 目标 1：用户有紧急沟通需求时不被阻断。
- 目标 2：高情绪价值内容通过等待机制放大体验。
- 目标 3：避免“全即时”稀释差异化，避免“全延迟”导致流失。

核心原则：

- 默认可用：任何时刻都可发即时消息。
- 默认有仪式：关键内容优先引导走马车信封。
- 规则透明：用户发送前可预期到达时间、费用（能量）与结果。

## 2. 双通道定位

### 2.1 即时通道（Instant Channel）

用途：

- 日常短沟通：问候、确认、行程同步、临时提醒。
- 情绪低负担互动：表情、短句、连续对话。

支持内容（MVP）：

- 文字（<= 300 字）
- 表情
- 已读状态

不支持内容（MVP）：

- 长语音、长文卡片、仪式化模板（统一放入马车信封）

### 2.2 马车信封（Carriage Envelope）

用途：

- 高情绪价值内容：告白、和解、纪念日寄语、睡前长文、精选照片。
- 需要“期待感”和“到达时刻感”的消息。

支持内容（MVP）：

- 文字（<= 1200 字）
- 语音（<= 60 秒）
- 单图（1 张）
- 模板信纸（基础 3 款）

体验特性：

- 显示预计到达时间（ETA）。
- 途中触发轻量事件（不改变消息语义，仅改变旅程情绪反馈）。
- 到达后触发专属提醒（振动 + 音效）。

## 3. 路由规则与交互策略

## 3.1 发送页默认策略

- 默认入口为“即时发送”，保证可用性。
- 当识别到高价值内容时，展示“升级为马车信封”建议卡。

触发建议卡的条件（任一命中）：

- 文本长度 > 120 字
- 附带语音
- 附带图片
- 命中节日/纪念日时间窗（当天或前后 24 小时）

## 3.2 反滥用与体验保护

- 马车信封设置最短旅程时间：3 分钟（避免被当成即时消息壳子）。
- 马车信封设置最长旅程时间：24 小时（避免体验失联）。
- 加急功能（MVP 可灰度）：
  - 每日 1 次免费加急，将剩余时间缩短 30%。
  - 加急后仍需保留至少 2 分钟等待时间。

## 3.3 会话层展示规则

- 即时消息：标准气泡流。
- 马车信封：展示“出发中/途中/已送达”状态卡片。
- 点击信封状态卡可进入“旅程页”查看 ETA、事件和里程进度。

## 4. 关键文案与行为约束

文案目标：明确分工，避免用户误解。

- 即时通道提示：`适合临时沟通，消息即时送达`
- 马车信封提示：`适合想被认真等待的话`
- 高价值升级引导：`这条消息很特别，要不要坐马车慢慢送到 TA 手里？`

行为约束：

- 不允许把马车信封默认设为全局发送方式。
- 不允许把即时消息改造成“假延迟”模式。

## 5. 事件与能量关系（MVP 最小闭环）

- 发送马车信封：+5 能量
- 接收并开启马车信封：+5 能量
- 旅程随机事件（概率触发）：+1~3 能量或视觉特效，不扣减基础奖励
- 即时消息：默认不产能量（防止刷量）

## 6. 指标与验收标准

MVP 关键指标（双周观察）：

- `马车信封渗透率`：至少 35% 的双人关系在首周发送过 >= 1 封马车信封
- `高价值内容转化率`：命中升级条件的消息中，>= 25% 选择马车信封
- `即时通道可用性`：即时消息发送成功率 >= 99.9%
- `信封完投率`：马车信封送达成功率 >= 99%

验收标准（功能）：

- 可在同一会话内稳定发送两类消息并正确展示状态。
- 发送页可触发升级引导且用户可一键切换。
- 旅程页 ETA 与会话卡片状态一致，无跨端错乱。
