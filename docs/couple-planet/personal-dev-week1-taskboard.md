# 情侣星球 Week1 每日任务看板（个人开发）

## 使用方式

- 每天只做当天清单，不跨天加功能。
- 每晚只回答两个问题：`今天是否完成`、`是否影响明天`。
- 任何新想法先记到“下周候选”，Week1 不实现。

---

## Day 1（后端基础与认证）

目标：跑通注册、登录、鉴权。

- [ ] 初始化后端项目（NestJS 或你熟悉栈）
- [ ] 建 `users` 表
- [ ] 实现 `POST /auth/register`
- [ ] 实现 `POST /auth/login`
- [ ] 实现 `GET /auth/me`
- [ ] 本地自测 3 条接口（Postman/curl）

完成标准：

- 可以注册新用户并拿到 token
- token 可访问 `GET /auth/me`

---

## Day 2（情侣绑定）

目标：跑通邀请码绑定链路。

- [ ] 建 `relations` 表
- [ ] 实现 `POST /relations/invite-code`
- [ ] 实现 `POST /relations/bind`
- [ ] 实现 `GET /relations/current`
- [ ] 处理“已绑定不可重复绑定”校验

完成标准：

- 两个账号能成功绑定为一对
- 重复绑定返回明确错误

---

## Day 3（即时消息后端）

目标：跑通文字消息发送/拉取/已读。

- [ ] 建 `messages` 表
- [ ] 实现 `POST /chats/current/messages`
- [ ] 实现 `GET /chats/current/messages`
- [ ] 实现 `POST /chats/current/messages/read`
- [ ] 增加发送幂等键（如 `clientMsgId`）

完成标准：

- 发送后可拉取到消息
- 已读状态可写入并查询

---

## Day 4（前端登录与绑定）

目标：前端接通 Day1-2 接口。

- [ ] 登录/注册页 UI
- [ ] 登录态持久化（token 存储）
- [ ] 绑定页 UI（邀请码输入）
- [ ] 绑定成功后跳转聊天页
- [ ] 错误提示文案（登录失败/绑定失败）

完成标准：

- 真机可完成“注册 -> 登录 -> 绑定”

---

## Day 5（前端聊天页）

目标：前端接通即时聊天链路。

- [ ] 聊天页 UI（消息列表 + 输入框）
- [ ] 接 `GET /chats/current/messages`
- [ ] 接 `POST /chats/current/messages`
- [ ] 发送中/发送失败状态显示
- [ ] 已读状态展示（最简）

完成标准：

- 两台设备可互发文字并看到消息

---

## Day 6（联调与异常修复）

目标：修掉主链路阻断问题。

- [ ] 双机联调完整走一遍（登录->绑定->聊天）
- [ ] 弱网场景测试（断网/重连）
- [ ] 消息失败重发
- [ ] 去重校验（避免重复消息）
- [ ] 修复阻断 bug（P0）

完成标准：

- 主链路无阻断
- 失败消息可重发且不重复

---

## Day 7（收尾与复盘）

目标：冻结 Week1 并决定是否进入 Week2。

- [ ] 回归测试一轮
- [ ] 记录 2 个核心数据
- [ ] 写 1 页复盘（问题/风险/下周重点）
- [ ] 明确 Week2 是否启动

核心数据（只看这两个）：

- 绑定完成率
- 即时消息发送成功率

完成标准：

- 形成可复用的 Week1 稳定版本
- 有明确 Week2 开始条件

---

## Week1 通过条件（Go）

- [ ] 两个账号可稳定注册、绑定、聊天
- [ ] 即时消息发送成功率 >= 99%
- [ ] 无 P0 阻断问题

若不满足：

- 不进入 Week2，先补稳定性与错误处理。
