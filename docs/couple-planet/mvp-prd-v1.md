# 情侣星球 MVP PRD v1

## 1. 文档目的

本 PRD 用于统一产品、设计、客户端、服务端与测试的 MVP 范围，覆盖：

- 双通道通讯（即时通道 + 马车信封）
- 星球主界面（关系状态与成长反馈）
- 离散定位 + 路径模拟技术方案
- 解绑冷静期与回忆封存机制

本版本以“可上线验证 PMF”为第一目标，不追求功能完整度。

## 2. 产品目标与北极星

### 2.1 目标用户

- 18-30 岁情侣用户（优先异地恋场景）

### 2.2 核心价值主张

- 即时沟通不受阻
- 高价值内容通过“等待”放大情绪价值
- 情侣互动可沉淀为可见、可回访的星球成长

### 2.3 北极星指标

- 首周马车信封渗透率（每对绑定关系至少发送 1 封）

## 3. MVP 范围

## 3.1 In Scope（必须上线）

1. 账号与双人绑定
2. 即时消息（文本/表情/已读）
3. 马车信封（文本/语音/单图、ETA、途中事件、到达提醒）
4. 星球页（状态展示、轻交互、能量与星屑反馈）
5. 解绑流程（72h 冷静期、撤销、封存、导出）

## 3.2 Out of Scope（明确不做）

- 3D 房间与家具系统
- 复杂星球建造编辑器
- 秒级实时轨迹追踪
- 社区/UGC

## 4. 核心用户场景

### 场景 A：日常短沟通

- 用户通过即时通道快速沟通，不引入等待成本。

### 场景 B：高情绪消息表达

- 用户发送马车信封，接收方在等待中感知“在路上”，到达后有仪式化打开体验。

### 场景 C：关系状态回访

- 用户打开星球页查看今日互动、在途状态、能量变化，形成回访动机。

### 场景 D：关系解除

- 用户发起解绑进入冷静期，支持撤销；生效后完成数据封存与导出。

## 5. 功能需求定义

## 5.1 双通道通讯规则

- 即时通道：
  - 支持文本（<=300 字）、表情、已读状态
  - 不产能量，避免刷量
- 马车信封：
  - 支持文本（<=1200 字）、语音（<=60 秒）、单图（1 张）
  - 最短 ETA 3 分钟，最长 24 小时
  - 触发建议卡条件：长文本、语音、图片、纪念日时间窗命中
  - 每日可免费加急 1 次（缩短剩余时间 30%，且保留至少 2 分钟）

## 5.2 星球页（MVP 主界面）

- 冷启动默认进入星球页
- 状态机：
  - `Dormant` / `Warm` / `Active` / `Bloom`
- 最小交互：
  - 点击星球（互动摘要）
  - 长按星球（关键时刻回放）
  - 上滑（7 日日志）
  - 点轨迹（跳转旅程页）
  - 点能量槽（能量流水）

## 5.3 能量与星屑闭环（最小版）

- 能量获取：
  - 发送马车信封 +5
  - 开启马车信封 +5
  - 当日首次互动奖励
- 能量消耗：
  - 信封加急
  - 高级信纸解锁
- 星屑：
  - 每封信封掉落星屑，仅展示累计值（暂不开放消耗）

## 5.4 定位与路径模拟

- 位置采样：发送前、接收缓存、低频修正（30 分钟）、到达前校准
- 不要求持续后台高精度定位权限
- 地图失败降级：城市级估算 + 固定速度模型
- 状态机：
  - `Created -> Dispatched -> InTransit -> Arrived -> Opened`
  - 异常分支：`Failed` / `Expired`

## 5.5 解绑与回忆封存

- 单方发起解绑后进入 72 小时冷静期
- 冷静期可撤销解绑
- 生效后：
  - 双方通信关闭
  - 共享资产进入封存仓
  - 支持导出（解绑后 30 天内）

## 6. 非功能要求

### 6.1 稳定性

- 即时消息成功率 >= 99.9%
- 马车信封送达成功率 >= 99%

### 6.2 性能

- 星球页关键交互反馈延迟 < 200ms（先动画后数据）
- ETA 计算接口 p95 < 300ms（缓存命中）

### 6.3 安全与隐私

- 定位最小化采集与存储
- 解绑/删除二次确认
- 关键流程写入审计日志（仅元数据）

## 7. 核心埋点与指标

### 7.1 埋点（关键）

- `message_send_instant`
- `message_send_carriage`
- `carriage_arrive`
- `planet_page_enter`
- `planet_state_changed`
- `unbind_apply`
- `unbind_cancel`
- `unbind_effective`

### 7.2 MVP 验证指标

- 首周马车信封渗透率 >= 35%
- 高价值内容转化率 >= 25%
- 星球页次日回访率 >= 45%
- 7 日留存 >= 40%

1. 用户与关系服务

3. 马车信封调度服务
4. 路径与 ETA 服务
5. 星球状态与资产服务
6. 解绑与封存服务

## 9. 版本计划（6-8 周）

- Week 1-2：
  - 账号、绑定、即时消息基础链路
- Week 3-4：
  - 马车信封创建、ETA、状态同步、旅程页
- Week 5-6：
  - 星球页状态机、轻交互、能量流水
- Week 7-8：
  - 解绑冷静期、封存导出、全链路压测、灰度上线

## 10. 依赖与风险

### 10.1 外部依赖

- 地图路径规划 API
- 推送服务（iOS/Android）

### 10.2 高风险项

- 地图服务波动导致 ETA 不稳定
- 弱网下状态错序
- 解绑与数据隔离的合规风险

## 11. 验收口径

- 功能验收：In Scope 全部可用，无阻断级缺陷
- 数据验收：关键埋点可回流并可看板化
- 体验验收：用户能清晰理解双通道差异与星球成长反馈
