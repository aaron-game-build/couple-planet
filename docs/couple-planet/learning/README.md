# Couple Planet 学习专区

本目录用于沉淀面向项目 owner 的产品、设计、流程与决策学习文档。

定位：

- 记录关键反思与复盘，不只记录“做了什么”，更记录“为什么这样做”
- 把经验沉淀为可复用方法，减少重复踩坑
- 将 AI 产出的讨论内容转为项目内可追溯资产

当前文档：

- `ai-product-design-reflection.md`：AI 参与下的产品与研发反思、问题成因与规避原则

建议维护规则：

- 每个迭代至少新增或更新 1 篇学习文档
- 每篇文档必须关联至少 2 份项目真源文档（PRD、API 契约、回归证据等）
- 复盘结论应可转化为流程门禁或模板字段
